# 🏠 AlmaHome — стартовый пакет

Привет, я завтра. Тебе сегодня лень разбираться, поэтому вот что в этой папке:

## Файлы в порядке использования

1. **`README.md`** — этот файл, читаешь первым
2. **`scripts/scrape-almahome.sh`** — скачивает старый сайт со всеми картинками
3. **`scripts/polygon-helper.html`** — инструмент чтобы разметить квартиры на планах этажей
4. **`CLAUDE.md`** — главный документ архитектуры. **Кладёшь его в корень нового Astro-проекта**, Claude Code будет читать его как контекст
5. **`PROMPTS.md`** — пошаговые промпты для Claude Code, этап за этапом

## Что делаешь когда пришёл домой

### Шаг 1. Скачать контент со старого сайта (5-10 минут)

```bash
# Если нет wget:
brew install wget

cd /путь/к/этой/папке/scripts
chmod +x scrape-almahome.sh
./scrape-almahome.sh
```

Через 10 минут у тебя в `scripts/almahome-old-content/` лежит весь старый сайт. Картинки квартир обычно в `images/` или `media/` — поскольку у тебя нет креды к Joomla, заходи на сайт через личный кабинет заказчика и сделай скриншоты ключевых страниц параллельно (главная, список квартир, детальная квартиры) — это сильно поможет потом.

### Шаг 2. Разметить планы этажей (30-40 минут)

Тебе нужны три картинки — план 1, 2 и 3 этажа. Они есть на старом сайте в скачанных ассетах. Если нет — попроси заказчика прислать.

```bash
open scripts/polygon-helper.html
```

Открывается в браузере:
1. Загружаешь картинку плана этажа
2. Вводишь номер квартиры → "Начать обводку"
3. Кликаешь по углам квартиры по часовой стрелке (4-6 точек хватает)
4. "Завершить квартиру" → переходишь к следующей
5. Когда все квартиры этажа размечены → "Скопировать JSON"
6. Сохраняешь куда-нибудь (текстовый файл, Notes), повторяешь для 2-3 этажа

В итоге — три блока JSON с координатами полигонов. Они потом пойдут в `src/data/floor-plans.json`.

### Шаг 3. Создать GitHub repo и аккаунты на сервисах (15 минут)

Параллельно с разметкой планов делай это:

1. **GitHub:** создай приватный repo `almahome`
2. **Vercel:** зарегистрируйся на vercel.com через GitHub (бесплатно)
3. **Resend:** зарегистрируйся на resend.com (бесплатно), верифицируй домен `almahome.lv` (нужен доступ к DNS заказчика, можно отложить — до этого слать с `onboarding@resend.dev`)
4. **GitHub Personal Access Token:** Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new → Scope: `repo` (полный) → создай и сохрани токен в надёжном месте

### Шаг 4. Инициализация проекта и Claude Code (10 минут)

```bash
mkdir ~/projects/almahome && cd ~/projects/almahome

# Скопируй сюда CLAUDE.md и PROMPTS.md из этой папки
cp /путь/к/заготовкам/CLAUDE.md .
cp /путь/к/заготовкам/PROMPTS.md .

git init
git remote add origin git@github.com:ТВОЙ_USERNAME/almahome.git

# Создай .env файл (потом заполнишь)
touch .env
echo ".env" >> .gitignore

# Запусти Claude Code
claude
```

### Шаг 5. Идёшь по PROMPTS.md этап за этапом (3 рабочих дня)

В `PROMPTS.md` 11 этапов с готовыми промптами. Открываешь промпт, копируешь, отправляешь Claude Code, ждёшь, проверяешь, идёшь дальше.

**Важно:** не пытайся ускорить процесс склеивая промпты. Каждый этап — отдельная итерация с проверкой в браузере. Если что-то не работает — разбираешься на этом этапе, не идёшь дальше с поломанным кодом.

После каждого работающего этапа — `git add . && git commit -m "Stage N: ..."` чтобы был промежуточный сейв.
